## Purpose

The Knowledge Circle tends the knowledge graph as the network's collective memory and constitutional oracle. They ensure the graph accurately represents network state, resolve entity disputes, maintain schema integrity, and coordinate federation with partner networks.

## Composition & Participation

- **Open to:** All [[Participant|Participants]], [[Member|Members]], [[Steward|Stewards]], and authorized [[Agent|Agents]]
- **Minimum:** 2 active participants with Member-level graph curation rights
- **Lead:** Optional facilitator role, rotating, selected by circle members
- **Agent Integration:** Agents have enhanced participation for graph operations

## Functions

| Area | Activities |
|------|-----------|
| **Graph Maintenance** | Entity creation, relationship management, schema updates |
| **Entity Reconciliation** | Resolve duplicates, manage merges, maintain references |
| **Query Patterns** | Maintain constitutional query library, document new patterns |
| **Federation Sync** | Coordinate with partner network graphs via MCP |
| **Documentation** | Ensure graph structure and usage is documented |

## Decision Authority

| Decision Type | Authority | Process |
|--------------|-----------|---------|
| **Entity operations** | Circle autonomy | Per [[3. Protocols/Asset Protocols/Knowledge Graph Protocol|Knowledge Graph Protocol]] |
| **Schema changes** | [[Network Assembly]] | 3-member consent + 48h window |
| **Federation agreements** | [[Network Assembly]] | Per [[3. Protocols/Asset Protocols/Federation Protocol|Federation Protocol]] |
| **Disputed edits** | Steward moderation | Per Knowledge Graph Protocol |
