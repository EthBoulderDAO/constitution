This protocol governs how someone becomes an Attendee and what that entails.

## Entry

### Requirements

| Requirement | Detail |
|-------------|--------|
| **Event attendance** | Attend an ETH Boulder event (annual convening, workshop, meetup) |
| **Verification** | Check-in at event or wallet signature |

### Process

| Step | Action | Actor |
|------|--------|-------|
| 1 | Attend ETH Boulder event | Person |
| 2 | Check in / verify attendance | Event Lead or system |
| 3 | Receive Attendee status | Automatic |
| 4 | Access Knowledge Graph (write) | Immediate |

No consent process required. Event attendance is the single entry membrane.

---

## Rights

| Right | Description |
|-------|-------------|
| **Graph write** | Create entities, add relationships |
| **Community participation** | Join discussions, activities |
| **Event access** | Participate in future events |
| **Membership eligibility** | Can be nominated for Member |

---

## Responsibilities

| Responsibility | Description |
|----------------|-------------|
| **Respect** | Treat all participants with respect |
| **Good faith** | Engage constructively |
| **Graph integrity** | Contribute accurate information |
| **Event conduct** | Follow event guidelines |

---

## Progression to Member

| Step | Action | Actor |
|------|--------|-------|
| 1 | Demonstrate ongoing participation | Attendee |
| 2 | Express interest in membership | Attendee |
| 3 | Receive nomination | Existing Member |
| 4 | 48h consent window | Member Assembly |
| 5 | Become Member | Upon consent |

## Verification

| Platform | Mechanism |
|----------|-----------|
